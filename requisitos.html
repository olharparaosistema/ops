Inclusão de requisitos

  • Objetivo: Simular o software no sistema e impactos conhecidos/previstos
  • Tempo estimado: 1 hora
  • Entrada: Mapa do sistema + Requisitos Funcionais
  • Saída: Mapa do sistema com software
  • Materiais necessários: Mapa do sistema; Material de desenho

Para complementar o mapa do sistema, é necessário ter em mãos o conjunto de requisitos funcionais do software. Os requisitos são apresentados (com slides ou lista) e lidos pelo mediador enquanto a posição de tais requisitos no mapa do sistema é discutida: O software (definido pelos requisitos) irá causar alterações e criações de elementos, relacionamentos e mais (Glossário de Conceitos Sistêmicos). Como estes efeitos da utilização do software são imediatos, não são complicados de identificar. Os efeitos imediatos em geral são o propósito do software sendo alcançado. Para o mapa, é ideal que eles sejam contrastados com o sistema atual, por exemplo, utilizando outra cor.
