Inclusão do software

  • Objetivo: Simular o software no sistema e impactos conhecidos/previstos
  • Tempo estimado: 30 minutos
  • Entrada: Modelo dinâmico do sistema (DSRP)
  • Saída: Modelo total do sistema (DSRP + simulação do software)
  • Materiais necessários: Requisitos do software

Para complementar o modelo do sistema, é necessário ter em mãos o conjunto de requisitos funcionais do software. Os requisitos são apresentados (com slides ou lista) pelo mediador enquanto a posição de tais requisitos no mapa do sistema é discutida:
O software (definido pelos requisitos) irá causar alterações e criações de elementos e relações. Como estes efeitos da utilização do software são imediatos, não são complicados de identificar. Os efeitos imediatos em geral são o propósito do software sendo alcançado. Para o modelo, é ideal que eles sejam contrastados com o sistema atual, por exemplo, utilizando outra cor.
