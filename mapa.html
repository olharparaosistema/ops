Construção do mapa

  • Objetivo: Disponibilizar o sistema de forma compreensível para análise
  • Tempo estimado: 1 hora
  • Entrada: Percepções
  • Saída: Mapa do sistema
  • Materiais necessários: Glossário de Conceitos Sistêmicos; Heurísticas; Material de desenho

Há um problema em discutir sistemas usando apenas palavras. Palavras devem vir uma de cada vez em ordem lógica e linear, já os sistemas acontecem inteiros ao mesmo tempo. Com figuras, todas as partes são vistas de uma vez.

- Thinking in systems: A primer

Precisamos visualizar o sistema para começar a entender do que ele é composto e como funciona!
O mapa de sistema nada mais é do que uma figura que representa o sistema graficamente, e é preciso uma análise inicial para construí-lo. Como ele pode se estender, o ideal é usar um quadro ou papel grande. 12 perguntas heurísticas apresentadas no link acima podem ser usadas para iniciar a montagem identificando as partes mais centrais ao software e portanto componentes importantes do sistema.

Os participantes devem identificar elementos e relacionamentos que compõe o sistema atual. Em seguida os itens identificados são representados graficamente no mapa de sistema que fica visível para todos durante as demais atividades. Mais conceitos sistêmicos além dos elementos e relacionamentos (que são básicos) podem ser encontrados no Glossário e utilizados se forem identificados.
