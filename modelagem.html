Modelagem DSRP

  • Objetivo: Aplicar a linguagem Systems Thinking no modelo
  • Tempo estimado: 1 hora
  • Entrada: Modelo de partes
  • Saída: Modelo dinâmico do sistema (partes + relações)
  • Materiais necessários: Padrões DSRP

Os participantes da análise precisam entender o idioma dos sistemas para ser possível modelar o sistema que receberá o software. Para isso, os padrões DSRP serão usados. O DSRP foi proposto por Derek Cabrera em 2015, e se trata de uma simplificação dos conceitos tradicionais de Systems Thinking. Assim é possível modelar o sistema de forma concisa focando apenas em 4 ideias essenciais. A descrição dos padrões está em Padrões DSRP.

O padrão D (Distinção) já foi incorporado ao modelo quando houve a definição de fronteiras do sistema e fronteiras entre as partes.

O padrão S (Estrutura) também já foi incorporado ao modelo quando houve a definição das partes e suas variáveis de interesse.

O padrão R (Relação) é o primeiro padrão que será incluído nessa etapa e provavelmente a atividade que irá demandar mais tempo e mais cognição. Para começar a desenhar a dinâmica do sistema, deve ser identificado como cada variável influencia outras (de qualquer parte). É preciso identificar e registrar no modelo não só a relação, mas também deixar representado se a ela é inversamente proporcional e em qual direção acontece (unidirecional ou bidirecional).
